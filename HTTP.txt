http是超文本传输协议，是网络传输数据的应用层协议，通常运行在TCP之上。
http有三个特点：
1.无连接：在http1.1之前经历一次请求-响应后链接就会关闭，现在可以通过请求头字段Connection：Keep-alive来进行长连接。
2.无状态：http协议本身不会记录客户端的状态，后续请求无法复原之前的上下文，可以通过session和cokie机制来进行弥补。
3.请求-响应模式：客户端发送请求，服务器回复响应，响应包含状态码（展开）。

http的无状态性
HTTP协议是无状态的（stateless）。也就是说，同一个客户端第二次访问同一个服务器上面的页面时，服务器无法得知这个客户端曾经访问过，服务器无法辨别不同的客户端。HTTP的无状态性简化了服务器的设计，是服务器更容易支持大量并发的HTTP请求。HTTP协议是采用请求-响应的模型。客户端向服务端发送一个请求报文，服务端以一个状态作为回应。当使用普通模式，即非keep-alive模式时，每个请求-应答都要重新建立一个连接，连接完成后立即断开；

HTTP1.1 使用持久连接keep-alive，所谓持久连接，就是服务器在发送响应后仍然在一段时间内保持这条连接，允许在同一个连接中存在多次数据请求和响应，即在持久连接情况下，服务器在发送完响应后并不关闭TCP 连接，而客户端可以通过这个连接继续请求其他对象。

HTTP 长连接不可能一直保持，例如 Keep-Alive: timeout=5, max=100，表示这个TCP通道可以保持5秒，max=100，表示这个长连接最多接收100次请求就断开。HTTP 是一个无状态协议，这意味着每个请求都是独立的，Keep-Alive 没能改变这个结果。另外，Keep-Alive也不能保证客户端和服务器之间的连接一定是活跃的，在 HTTP1.1 版本中也如此。唯一能保证的就是当连接被关闭时你能得到一个通知，所以不应该让程序依赖于 Keep-Alive 的保持连接特性，否则会有意想不到的后果。

http的常见状态码：
1xx：表示服务器已经接收到客户端请求，客户端可以继续发送请求；
2xx：表示服务器已经成功接收请求并处理；
3xx：表示服务器要求客户端重定向；
4xx：客户端请求有问题；
5xx：服务器未能正常处理客户端的请求发生错误；

·200 OK：表示请求已成功，请求所期望的响应头或数据体将随之返回。例如，当客户端发送一个获取网页的GET请求，若服务器成功返回网页内容，就会返回200OK。
201 Created：表示请求已成功，且服务器创建了新的资源。此状态码通常在POST请求成功创建资源时返回，响应头中会包含新资源的URL（Location字段）。
204 No Content：说明服务器成功处理了请求，但不需要返回响应体。常用于 DELETE 请求成功删除资源后，或者某些 PUT 请求成功更新资源但无需返回特定数据时。

301 Moved Permanently：永久性重定向，表明请求的资源已被永久移动到新的 URL。搜索引擎会更新索引，将旧 URL 替换为新 URL。客户端后续请求应使用新的 URL。例如，网站域名更换，可返回 301 让浏览器自动跳转到新域名。
302 Found：临时性重定向，请求的资源临时位于不同的 URL。客户端应继续使用原 URL 进行后续请求。与 301 不同，搜索引擎不会更新索引。

·400 Bad Request：表示客户端请求语法有问题，不能被服务器理解。
·401 Unauthorized：意味着客户端需要进行身份验证，但尚未提供有效的身份凭证。常见于需要登录才能访问的资源，未登录或登录失败时返回此状态码。
·403 Forbidden：表示服务器收到请求但拒绝提供服务，通常会在响应正文中给出不提供服务的原因。
·404 Not Found：请求的资源不存在，可能是URL输入错误，或者资源已被删除。

500 Internal Server Error：服务器发生错误，无法完成客户端请求；
503 Service Unavailable：表示服务器当前不能处理客户端请求，一段时间之后可能恢复正常；
